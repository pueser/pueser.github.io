## ◆ Git 명령어 정리

---

## 1. Git에 처음 코드 올리기

|순서|명령어|설명|
|---|---|---|
|1|`git init`|현재 폴더를 Git 저장소로 초기화|
|2|`git add .`|모든 변경 파일을 Stage에 추가|
|3|`git commit -m "Initial commit"`|첫 번째 Commit 생성|
|4|`git remote add origin <GitHub_URL>`|원격 저장소 연결|
|5|`git push -u origin main`|GitHub에 최초 업로드 (`-u`는 기본 브랜치 등록)|


> `-u` 옵션을 사용하면 이후부터는 `git push`만 입력해도 됩니다.

---

<br>
<br>

## 2. 저장소 복제(Clone)

|명령어|설명|
|---|---|
|`git clone <GitHub_URL>`|GitHub 저장소를 로컬 PC로 복제|

---

<br>
<br>

## 3. 변경 사항 확인

|명령어|설명|
|---|---|
|`git status`|현재 변경된 파일 확인|
|`git diff`|수정된 내용 확인|
|`git diff --staged`|Stage에 추가된 내용 확인|
|`git log`|Commit 내역 확인|
|`git log --oneline`|Commit을 한 줄로 간략하게 표시|

---

<br>
<br>


## 4. 파일 추가(Add)

|명령어|설명|
|---|---|
|`git add 파일명`|특정 파일만 Stage에 추가|
|`git add .`|현재 폴더의 모든 변경사항을 Stage에 추가|

---

<br>
<br>


## 5. Commit

|명령어|설명|
|---|---|
|`git commit -m "메시지"`|Stage에 있는 파일을 Commit|
|`git commit --amend`|마지막 Commit 메시지 또는 내용 수정|

---

<br>
<br>


## 6. Push

|명령어|설명|
|---|---|
|`git push`|현재 브랜치를 원격 저장소에 업로드|
|`git push origin main`|main 브랜치 업로드|
|`git push -u origin main`|최초 Push 및 기본 브랜치 등록|
|`git push --force`|강제 Push (주의)|

---

<br>
<br>


## 7. Pull

|명령어|설명|
|---|---|
|`git pull`|원격 저장소 변경사항 가져오기 + Merge|
|`git pull origin main`|main 브랜치 가져오기|

---

<br>
<br>


## 8. 원격 저장소(Remote)

|명령어|설명|
|---|---|
|`git remote -v`|등록된 원격 저장소 확인|
|`git remote add origin URL`|원격 저장소 등록|
|`git remote remove origin`|원격 저장소 삭제|
|`git remote set-url origin URL`|원격 저장소 URL 변경|

---

<br>
<br>


## 9. Commit 취소(Reset)

### 최근 Commit 취소

|명령어|설명|
|---|---|
|`git reset --soft HEAD~1`|Commit만 취소 (Stage 유지)|
|`git reset --mixed HEAD~1`|Commit + Stage 취소|
|`git reset --hard HEAD~1`|Commit + Stage + 작업내용 모두 삭제|

<br>

### 원격 저장소 상태로 되돌리기

|명령어|설명|
|---|---|
|`git reset --soft origin/main`|Commit만 취소|
|`git reset --mixed origin/main`|Commit + Stage 취소|
|`git reset --hard origin/main`|원격 저장소와 동일한 상태로 복원|

> **⚠️ `reset --hard`는 변경사항이 모두 삭제되므로 주의해서 사용하세요.**

## 10. Stash (작업 임시 저장)

|명령어|설명|
|---|---|
|`git stash`|현재 작업을 임시 저장|
|`git stash list`|임시 저장 목록 확인|
|`git stash pop`|작업 복원 후 Stash 삭제|
|`git stash apply`|작업만 복원 (Stash 유지)|

---

<br>
<br>


## 11. Pull 충돌(Conflict) 발생 시

|순서|명령어|설명|
|---|---|---|
|1|`git stash`|현재 작업 임시 저장|
|2|`git pull origin main`|최신 코드 가져오기|
|3|`git stash pop`|내 작업 복원|
|4|충돌 해결 후 `git add`, `git commit`|충돌 해결 및 Commit|

## 12. Git 사용자 정보 설정

### 전역(Global) 설정

> 모든 Git 프로젝트에 적용됩니다.

|명령어|설명|
|---|---|
|`git config --global user.name "이름"`|사용자 이름 설정|
|`git config --global user.email "메일"`|사용자 이메일 설정|
|`git config --list`|현재 설정 확인|



<br>


### 현재 프로젝트(Local) 설정

> 현재 프로젝트에만 적용됩니다.

|명령어|설명|
|---|---|
|`git config user.name "이름"`|사용자 이름 설정|
|`git config user.email "메일"`|사용자 이메일 설정|
|`git config --list`|현재 설정 확인|

## 13. 브랜치(Branch)

|명령어|설명|
|---|---|
|`git branch`|브랜치 목록 확인|
|`git branch 브랜치명`|브랜치 생성|
|`git switch 브랜치명`|브랜치 이동 (권장)|
|`git switch -c 브랜치명`|브랜치 생성 후 이동|
|`git checkout 브랜치명`|브랜치 이동 (구버전 방식)|
|`git checkout -b 브랜치명`|브랜치 생성 후 이동 (구버전 방식)|
